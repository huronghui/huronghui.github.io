android studio 使用Maven 仓库

因为项目需要，我们要用自己的maven库来构建系统，这就需要我们在 Android Studio 中配置maven 仓库，其实在我们的使用中，我们已经在不经意间使用了 maven 的库。

###什么是maven
对于我们android 开发者而言，我们只需要知道 Maven 是一个构建工具， Maven 包是由 POM（project Object Model）所定义的文件格式
Gradle 是可以使用 Maven 包的，并且大部分的远程依赖都是 Maven 包，使用这种包的方式，我觉得大家应该都是清楚的：

###Maven 仓库
Maven包存放的地方，就是 Maven仓库了，这些仓库，可以是放在本地，也可以是放置在远程服务器上，当然也可以是私有仓库，也可以是公开的，其实在我们的开发中，我么已经在 Android Studio 中配置了 Maven 仓库，看下面代码：

其中的 jcenter 就是一个Maven 仓库，jcenter 和 mavenCentral() 是 Android Studio 主要支持的两个 Maven 仓库，其中 MavenCentral 是最早的 maven 中央仓库，而jcenter 是 Android Studio 0.8版本开始的默认中央仓库。

根据这篇博客，jcenter 具有以下的特点：

  1. jcenter 是一个 mavenCentral 的超集，jcenter 还包含了其他 maven 包
  2. jcenter 具有更好的 cdn，默认支持 https，这个对于谷歌有巨大吸引力
  3. bintray（jcenter 的服务提供商）表示 jcenter 具有更好的性能
  4. 有数据表明 bintray jcenter 占用更少的本地缓存空间
  5. 更好的交互界面，可以从 jcenter 向 mavenCentral 同步包（下面两个平台的使用教程将会证实这一点）
